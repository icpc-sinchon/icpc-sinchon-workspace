# 규칙

## 등록

- 팀원 전원이 도착해야 등록이 가능합니다. (단, 등록 전 행사장 내 대기는 가능)
- 이름과 사진을 포함한 신분증(주민등록증, 운전면허증, 학생증 등)을 지참하여야 합니다.
- 팀 레퍼런스 문서를 지참하는 경우 등록 시 제출하여 검사받아야 합니다.

## 팀 레퍼런스 문서

- 대회 중 인터넷 검색은 불가합니다.
- 표지를 제외하고 단면 A4 기준 최대 25쪽까지 허용됩니다. 출력물 형태만 가능하며, 동일 문서는 팀당 최대 3권(총 75페이지)까지 지참할 수 있습니다.
- 팀 레퍼런스 문서는 등록 시 검사하며, 반려 시 지참할 수 없습니다.
- 규격 미달 혹은 검사 시 반려된 문서는 대회 중 사용할 수 없습니다.
- 글자 크기가 지나치게 작거나(<2mm, 약 6pt), 타 팀이 육안으로 식별 가능할 정도로 큰 경우 반려 사유가 됩니다.
- 표지를 제외한 모든 페이지에는 페이지 번호가 인쇄되어 있어야 합니다.

## 프로그래밍 도구

- 모든 코드는 대회 시간 내에 직접 작성해야 합니다.
  - 팀 레퍼런스 문서의 내용을 참조하여 작성하는 것은 허용됩니다.
  - 허용된 공식 언어 레퍼런스 웹사이트의 코드는 복사하여 사용할 수 있습니다.
  - 사전 작성된 템플릿 코드나 외부 라이브러리(예: Atcoder Library)는 사용할 수 없습니다. 이는 제출 코드뿐만 아니라 테스트용 코드에도 동일하게 적용됩니다.
- **Github Copilot, lex, yacc 등 코드 생성 플러그인 및 ChatGPT 등 생성형 AI 사용은 엄격히 금지됩니다.**
  - LSP-Clangd, IntelliSense 등 기본 자동완성 기능은 사용 가능합니다.
  - Code Runner, Competitive Programming Helper 등 코드 작성을 보조하지 않는 도구는 사용 가능합니다.
  - 코드 작성과 관련되지 않은 기타 도구(단축키를 사용하여 컴파일, 입력의 파이프라인, Unix의 `diff` 등 코드의 단순 비교 등)는 사용 가능합니다.
- 관련 기타 문의 사항이 있는 경우 아래의 카카오톡 채널을 통해 문의해 주세요.
